# BidRadar-X 工作计划

更新时间：2026-07-14（Asia/Shanghai）

## 1. 状态定义

- `已完成`：实现和验收证据均满足当前任务条件。
- `进行中`：已有人声明文件范围并开始工作。
- `待开始`：依赖已满足但尚未认领。
- `阻塞`：存在明确外部依赖，必须在独立任务日志中写明解除条件。
- `后续阶段`：不属于当前第一目标，禁止提前挤占关键链路。

## 2. 执行规则

- 当前第一目标固定为：真实数据源 → 清洗去重 → 增量判断 → DOCX。
- 多窗口只能修改各自声明的文件范围。开始前必须创建独立任务日志并写明负责人/窗口和文件范围。
- 每完成一个阶段，立即更新自己的独立日志；不得让另一个窗口代写或与其他任务共用日志。
- 不允许把账号、Cookie、Token、API Key 写入仓库。涉及登录或密钥的任务只能提交无敏感值的配置契约和变量名。
- 未经协调不得修改现有业务文件；本计划中的建议范围是认领边界，不是自动授权。

## 4. 阶段门禁

### 门禁 A：真实数据可用

完成 F01、F02、R01、R02、R03。验收不接受模拟 URL、手工复制结果或把解析路由名称当成解析能力。

### 门禁 B：数据可信可追溯

完成 N01、N02、D01。验收必须能从项目事实回到真实来源和具体原文位置。

### 门禁 C：增量成立

完成 I01、I02、I03。验收必须进行至少两次运行并证明幂等、变化依据和水位线恢复。

### 门禁 D：DOCX 可交付

完成 W01、W02、W03。验收必须从产品入口下载并打开真实数据生成的 DOCX。

### 门禁 E：比赛演示就绪

完成 Q01、Q02。只有通过该门禁，才允许将 L01—L03 从“后续阶段”提议为新阶段任务。
