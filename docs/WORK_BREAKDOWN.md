# 双人逐功能分工与 Agent 任务包

角色固定：A＝数据科学/产品负责人（你）；B＝软件工程/平台负责人（队友）；S＝必须双方审核。每个编号对应一个独立 GitHub Issue。

## 第0阶段：协作和契约

| ID | 负责人 | 小步 | 具体产出 | 验收 |
|---|---|---|---|---|
| S-01 | S | 确认范围 | P0/P1/P2清单、非目标、演示提示词 | 双方在 Issue 留 `approved` |
| B-01 | B | 开发环境 | `bootstrap.ps1`、`dev.ps1`、依赖锁 | 两台电脑从克隆到启动≤10分钟 |
| S-02 | S | 冻结契约 | JobSpec、TenderRecord、SourceAdapter、API v1 | 样例 JSON 可通过模型校验 |
| A-01 | A | 字段字典 | 字段含义、来源、空值、标准化规则 | 10条公告人工填写无歧义 |
| A-02 | A | 来源验证表 | 三个候选站点条款、robots、限速、30条小样 | 给出 approved/pending/disabled 结论 |

## 并行边界

- A 可并行开发分析逻辑，但只能依赖 `docs/CONTRACTS.md` 和 fixture，不等待 B 的后端。
- B 可并行开发平台，但不得内嵌相关性、去重或字段业务规则，统一调用 A 提供的纯函数接口。
- `S-*` 是合流门禁；未通过时不得进入下一阶段的共享主链。
- 两个 Agent 不得同时编辑同一公共契约文件。契约变更先合并，再分别 rebase。
