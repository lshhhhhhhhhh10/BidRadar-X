# BidRadar-X 报名表直接填写稿

更新日期：2026-07-12

## Part 1｜命题前置分析与洞察

> 建议字数：50–300字；以下正文不含标题和本说明。

我们将命题理解为：帮助企业从分散、异构且持续变化的公开招投标信息中，及时识别可信商机，而非简单制作一个爬虫。财政部数据显示，2024年全国政府采购规模达3.38万亿元；中国政府采购网等平台每天持续发布招标、更正、中标等公告。剑鱼标讯、千里马等产品已提供关键词订阅、多端推送和企业分析，说明“聚合+订阅”需求成熟，但通用平台仍难针对服务器、液冷、算力中心等专业场景解决同一项目跨站重复、公告版本变化、关键字段缺少原文证据等问题。因此本方案重点从“搜得多”升级为“找得准、看得懂、可核验、能追踪”。

## Part 2｜整体解决方案设计

> 建议字数：300–600字；以下正文不含标题和本说明。

BidRadar-X是一套面向服务器、算力中心及企业IT解决方案团队的招投标机会雷达。用户以自然语言描述地区、主题、公告类型、时间与频率；系统先解析为结构化JobSpec，无时间要求则立即执行，有明确频率则由调度器定时运行。采集层通过合规的SourceAdapter接入政府采购及公共资源交易平台，依次完成网页/PDF解析、字段标准化、相关性排序、跨站项目归并和版本识别，最终生成可下载的Word报告及任务记录。核心创新一是“字段级证据链”：预算、截止日期、采购人等关键字段均关联原文片段、来源链接、抓取时间和内容哈希，缺失信息标为“未知”，降低大模型幻觉风险；创新二是“项目变更雷达”：将预告、招标、更正、废标和中标公告归入同一项目时间线，定时报告优先呈现新增与变化，而非重复堆叠信息。系统采用FastAPI、APScheduler、SQLite和本地Web界面，模型仅用于意图解析、非关键字段辅助提取和摘要，抓取、核验、去重及任务状态均由可测试的确定性程序完成。试点目标是在人工核验样本上实现公开源抓取成功率不低于90%、相关结果Precision@10不低于80%、报告关键事实证据覆盖率100%，并将一次人工检索与整理流程压缩到15分钟以内。适配器与行业词库可配置，可从华东算力场景推广到其他地区和行业。
