# 报名前双人分工（当前最高优先级）

在本阶段，暂停业务代码和复杂架构。两人的 Agent 产出必须服务于 Part 1、Part 2 和补充材料。

| 步骤 | 负责人 | Agent输入 | 必须产出 | 验收规范 |
|---|---|---|---|---|
| R-01 公开事实包 | A | 官方题目、政府采购公开资料 | `research_pack.md`：每条结论、来源、日期、原文摘要、是否推断 | 不引用营销软文支持核心市场数字 |
| R-02 竞品矩阵 | A | 剑鱼、千里马等官方产品页 | 搜索/订阅/推送/分析/证据/变更追踪矩阵 | “没有某功能”必须写“公开页面未发现”，不得断言不存在 |
| R-03 用户访谈 | A主、B辅 | 统一访谈提纲 | 3–5份匿名记录和痛点排序 | 不编造反馈，不收集商业机密 |
| R-04 方案契约 | B | 官方要求、JobSpec/TenderRecord草案 | 一页架构、端到端流程、模块输入输出 | 每个模块能映射到官方要求或创新点 |
| R-05 技术可行性 | B | 三个候选来源 | 来源访问验证、风险、降级路径 | 不绕过验证码；样例与实时源分开 |
| R-06 量化指标 | A主、B审 | 金标集计划、系统链路 | 指标定义、分母、样本量、基线、目标 | 全部标注“目标/已测”，不得混淆 |
| R-07 文案合并 | S | R-01至R-06 | 300字内Part1、600字内Part2 | 字数合规、无重复、无未证实成绩 |
| R-08 补充材料 | S | 最终文案与证据 | 6–8页PDF或DOCX | 首两页可让评委理解问题、差异、落地价值 |

Agent 每次交付统一使用：完成内容、证据与链接、改动文件、验证、未决风险。任何无来源的市场数字、虚构访谈、虚构准确率一律退回。
