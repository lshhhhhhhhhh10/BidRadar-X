## 1. 结论

首个权威公开来源选定为**中国政府采购网**。其[官方首页](https://www.ccgp.gov.cn/)自述为“财政部唯一指定政府采购信息网络发布媒体”，页脚列明主办单位为中华人民共和国财政部国库司、网站标识码为 `bm14000002`，可作为权威发布位置使用。这里的“权威”不等于能够证明每条公告在该站首发；来源角色中的 `original` 只表示权威发布记录，不声明时间上的首发。

匿名访问可以完成“官方检索入口 → 公告详情 → 附件入口元数据”的首段链路，不需要账号或凭据。附件正文是否可匿名取得取决于每条公告：官方详情可能给出站外对象存储链接，也可能要求前往其他交易平台登录。F01 只确认并约束附件入口，不把未下载的附件写成已获取。

合规结论是**有条件可用**：本次未找到公开的自动化采集条款或速率上限，`www` 与 `search` 两个主机的 `robots.txt` 均返回 404。404 既不是允许，也不是禁止。生产规模启用前必须保留人工合规复核点，并采用本文件中比现有实现更保守的内部访问上限；若官方以后发布 robots、条款、停止提示或联系通知，以官方新要求为准。

### 4.1 证据定位契约

每条字段证据至少保存：

- `field_path`：稳定输出路径；
- `source_url`：详情页或已验证附件 URL；
- `fetched_at`：带时区的证据采集时间；
- `quote`：支持字段值的最短完整原文；
- `locator`：HTML CSS 选择器加表头/段落序号，例如 `公告概要表/预算金额`；
- 附件证据另存 `attachment_id`、文件名、页码或文档内定位（仅在附件实际解析后填写）。

标题、发布时间、正文、附件入口和来源 URL 也必须有定位记录。页面正文可以作为事实来源，但不能把整个 HTML 当作单个字段的定位。原文片段不得包含不必要的联系人号码或其他与目标字段无关的个人信息。

### 4.2 附件入口与下载边界

F01 把以下状态明确分开：

1. `discovered`：官方详情列出附件名称或目标；只保存入口。
2. `download_allowed`：目标主机、公开访问条件、类型和大小策略均通过；F01 不产生此状态。
3. `downloaded`：后续 R02 实际流式读取并校验；必须有内容哈希和采集时间。
4. `blocked/restricted/invalid`：需登录、验证码、CA、站外来源未批准、超限、类型异常或重定向异常。

自动下载的后续最低限制：

- 默认只允许 CCGP HTTPS 主机；站外对象存储或交易平台必须有独立来源审查和显式允许列表。
- 先校验最终重定向主机、声明类型和文件头；不得只信扩展名。
- 建议单文件硬上限 50 MB，流式读取，超限立即中止；该数值是 BidRadar-X 内部安全上限，不是官方配额。
- PDF、Office 文档可进入解析队列；压缩包先隔离并做解压大小、文件数、路径穿越和嵌套层数检查。
- 登录、CA、验证码或带签名凭据的下载一律停止，不从浏览器复制会话，不把完整查询串或响应头写日志。
- 附件失败只产生 `status=partial` 并附 `ATTACHMENT_RESTRICTED` 或 `ATTACHMENT_INVALID` 错误码，不能丢弃已成功记录的公告详情，也不能写成附件已取得。

## 6. 合规边界与停止条件

### 6.1 robots、条款和版权

- 两个 robots 路径的 404 只记录为“无法取得规则”；不得解释为允许全站抓取。
- 官方首页的财政部版权和主办信息用于确认来源身份，不等于授予批量复制、再分发或商业利用权。
- 未确认自动化条款时，只做满足产品首条链路的最小公开访问；上线规模、长期保存和再分发应由项目责任人复核适用法律与官方最新规则。
- 每次发布版本、至少每季度以及首次出现官方限制页时，重新核对 robots、页脚条款和联系通知。

### 6.2 明确禁止

- 禁止绕过验证码、安全验证、WAF、频控、IP 限制或访问拒绝。
- 禁止代理/IP 轮换、指纹伪装、冒充真人浏览器或降低请求可识别性。
- 禁止从个人浏览器导出 Cookie、会话、账号、CA、Token 或其他登录凭据。
- 禁止把凭据、完整会话头、签名下载 URL 或响应中的隐私数据写入仓库、日志、fixture 或错误对象。
- 禁止因解析失败使用列表摘要、旧缓存或模板伪造字段。
- 禁止自动跟随到未审查的站外附件主机；只记录入口和受限原因。

### 6.3 立即停止本次来源运行

出现任一条件时立即停止，不继续分页、不换入口、不重试绕过，且水位线不推进：

- HTTP 403、429，或页面出现访问过于频繁、验证码、安全验证、登录/CA 要求、明确停止文本；
- robots 或平台条款新增禁止规则；
- 官方通过页面、响应头或联系渠道提出停止；
- 搜索结果容器、详情标题/发布时间/正文结构无法确认；
- 重定向离开允许的 CCGP 主机；
- 响应类型异常、HTML 超过 10 MB、附件超限或文件头与声明类型冲突；
- 来源持续 5xx/超时达到熔断阈值。

## 7. 失败策略与结构化结果

```json
{
  "source_id": "ccgp",
  "stage": "search|detail|attachment",
  "status": "success|empty|partial|blocked|unavailable|parse_drift",
  "error_code": "stable_machine_code_or_null",
  "http_status": null,
  "attempts": 1,
  "retryable": false,
  "stop_reason": null,
  "watermark_advanced": false,
  "notice_count": 0,
  "attachment_count": 0,
  "fetched_at": "timezone-aware ISO 8601",
  "operator_action": null
}
```

错误结果只保存去除用户查询敏感片段后的 URL、稳定错误码和必要响应元数据；不保存 Cookie、Authorization、完整响应头、安全页表单值或附件签名参数。

| 失败 | 错误码/状态 | 重试与停止 | 数据副作用 |
| --- | --- | --- | --- |
| DNS 临时失败 | `DNS_ERROR` / `unavailable` | 最多 2 次退避；仍失败则来源熔断 | 不写成功缓存，不推进水位线 |
| 连接/读取超时 | `TIMEOUT` / `unavailable` | 最多 2 次退避；保持串行限速 | 已持久化公告不回滚，当前页不推进 |
| HTTP 429 | `HTTP_429` / `blocked` | 0 次重试；遵循 `Retry-After`，没有时人工复核 | 熔断来源，不推进水位线 |
| HTTP 403 | `HTTP_403` / `blocked` | 0 次重试；不换 UA/IP/会话 | 熔断来源，不推进水位线 |
| HTTP 200 安全页/验证码/登录页 | `SECURITY_CHALLENGE` / `blocked` | 0 次重试；立即停止 | 不把页面缓存为成功或空结果 |
| 其他 4xx | `HTTP_4XX` / `unavailable` | 不重试；404 可短负缓存 | 当前 URL 失败，水位线不推进 |
| 5xx | `HTTP_5XX` / `unavailable` | 最多 2 次退避；持续失败熔断 30 分钟 | 不写成功缓存，不推进水位线 |
| 搜索或详情结构变化 | `STRUCTURE_CHANGED` / `parse_drift` | 不重试；保留有限、去敏元数据供人工复核 | 不能改报 `empty`，不推进水位线 |
| 必填标题/发布时间/正文缺失 | `REQUIRED_FIELD_MISSING` / `partial` | 该详情不产出公告；达到阈值停止整源 | 已成功的其他公告保留，整页水位线不推进 |
| 附件站外/登录/异常类型/超限 | `ATTACHMENT_RESTRICTED` 或 `ATTACHMENT_INVALID` / `partial` | 不绕过；仅保留入口和原因 | 公告保留，附件不标记已下载 |
| 真实空结果 | `EMPTY_OK` / `empty` | 已确认结果容器存在且列表为空时正常结束 | 可记录成功运行；仅在全部页处理成功后推进水位线 |

## 9. F01 验收判定与后续边界

F01 可以判定完成，因为已经：

- 选定并用一手官方页面确认了一个权威公开来源；
- 核对了检索入口、真实详情字段和附件入口；
- 明确了字段、证据、访问、合规、停止和结构化失败契约；
- 记录了 robots、条款、官方速率和站外附件授权的无法确认项；
- 证明检索、详情和附件入口发现无需凭据，同时不把受限附件写成已下载；
- 将现有实现逐项分成已实现、需后续实现和无法确认，且没有用代码替代合规证据。

F01 完成不代表 R01、R02、F02 或端到端真实链路完成。后续实现必须以本契约为门禁；若无法满足 2 秒全局间隔、结构化失败、缓存/水位线和附件边界，不得宣称生产采集已合规。

## 10. 变更复核

下列事件任一发生时必须更新本文件并重新验收：

- 官方检索入口、参数、DOM、详情分类或附件标签变化；
- robots、平台条款、版权/数据再利用说明或官方联系通知变化；
- 首次真实出现 429、验证码、安全页、登录要求或新的站外附件主机；
- 调整频率、并发、缓存、附件大小、重试或水位线策略；
- 新增第二个来源或把 CCGP 页面之外的交易平台纳入自动采集。
