# BidRadar-X 项目上下文

更新时间：2026-07-19（Asia/Shanghai）

## 一句话目标

把分散、变化快、格式不统一的真实招投标公告，转换成可追溯、可持续更新、可解释并能下载为 Word 的投标机会。

## 当前产品链路

```text
自然语言输入
→ 需求结构化
→ 同义词与采购表达扩展
→ 多来源检索与详情/附件采集
→ 规范化、相关性复核、生命周期过滤与去重
→ 字段级事实证据
→ 单项目 Word + AI 辅助摘要/风险研判
→ 查询历史、收藏与增量订阅
```

AI 负责理解、语义复核和组织表达；固定程序负责网站访问、来源适配、事实字段、附件、指纹、预算、调度和 DOCX 发布。缺少模型凭据时系统可以规则降级，但不能把规则结果标记为 AI 成功。

## 已实现能力

- 自然语言主题、地区、时间和频率提取。
- AI 扩词、检索规划、相关性复核、歧义去重、事实核验和报告研判。
- CCGP、全国 GGZY、中国移动公开公告生产适配器。
- 天眼查和 SAM.gov 用户自有凭据适配器。
- 公告详情、附件入口、联系人和常见投标指标提取。
- 多来源状态、空结果与失败原因的诚实展示。
- SQLite 版本化迁移、报告/订阅持久化、快照、水位线和 Outbox。
- 查询历史、收藏、定时任务管理、单项目 Word 下载和结构化详情。
- Word 生成后回读验证，以及 AI 主备凭据、退避和规则降级。
- 每日付费来源预算上限的调用前原子拦截。

## 需凭据或外部条件

- 天眼查、SAM.gov 需要用户自己的 API Token/Key。
- 智谱 AI 需要用户自己的 API Key；额度与服务可用性由模型平台决定。
- 飞书多维表格投递底座已有代码和测试，但真实企业租户、字段映射和知识库权限尚未完成线上验收。

## 明确未完成

- TED 和中国招标投标协会未注册生产适配器。
- 企业知识库 RAG、企业资质等价判断、历史成交价格与利润模型尚未形成可验收产品。
- 飞书企业知识库招投标文件自动同步归档仍为下一阶段规划。
- 外部网站可用性不能保证 100%；系统能够限速、重试、隔离和展示原因，但无法控制第三方停机或策略调整。

## 当前技术架构

- `app/`、`lib/`：React/vinext 前端与 API 客户端。
- `backend/app/api/`：FastAPI 任务、来源、报告、订阅和 AI 状态接口。
- `backend/app/workflow/`：LangGraph 编排与五阶段工作流。
- `backend/app/sources/`：站点生产适配器注册表。
- `backend/app/ai/`：结构化 Prompt、模型客户端、审计和主备故障切换。
- `backend/app/intelligence/`：证据、指标、事实一致性与去重逻辑。
- `backend/app/storage/`：SQLite、迁移、仓储、快照、水位线和 Outbox。
- `backend/app/services/docx_publisher.py`：证据化 DOCX 生成与回读验证。
- `backend/app/services/scheduler*.py`：持久化定时任务、租约、重试和恢复。
