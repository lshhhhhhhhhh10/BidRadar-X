# 全国官方与上海官方招标公告来源研究

研究日期：2026-07-15（Asia/Shanghai）

## 结论摘要

第一阶段可以接入以下两个官方来源，但必须使用“来源原生分类 + 正文结构 + 有效时间 + 生命周期关联”的分层判定，不能只靠标题关键词：

1. 全国：中国招标投标公共服务平台公告公示栏目（`bulletin.cebpubservice.com`）。它原生分开了招标公告、资格预审公告、更正公告、中标候选人公示和中标结果公示。
2. 上海：上海市公共资源交易平台的国企采购栏目（`www.shggzy.com/gqcg`）。它原生提供“招标公告/资格预审公告”和“变更公告”栏目，并把候选人、结果和异常公告放在不同栏目。

不能把“更正/变更公告”栏目整体当作有效招标机会。全国平台的更正列表在本次实测中包含“流标公告”和“终止公告”；只有能关联到原招标/资格预审公告、且应用最新变更后仍存在未来报名、文件获取、资格预审申请或投标截止时间的公告，才能进入产品结果。

## 研究方法与证据边界

- 只使用平台自己的公开页面、公开前端脚本和平台链接到的详情应用；未使用第三方转载页面作为事实来源。
- 通过实时 HTTP 请求核验了列表页、分类控件、分页脚本和若干公开详情页。
- 未发现两个平台面向第三方公开承诺的稳定 API。下文列出的是网站自身当前使用的公开 HTML 路径/前端请求，不应被描述为有 SLA 的官方开放 API。
- 全国平台详情应用的内部只读接口触发了 JavaScript 反自动化挑战，未能可靠核验其 JSON 字段；相关部分按“不确定”处理。

## 1. 中国招标投标公共服务平台（全国）

### 1.1 官方入口与原生分类

官方公告入口：[中国招标投标公共服务平台公告公示](https://bulletin.cebpubservice.com/)

入口页面公开 HTML 中的分类 ID 和标签为：

| `categoryId` | 平台标签 | 本项目处理 |
|---|---|---|
| `88` | 招标公告 | 允许 |
| `92` | 资格预审公告 | 允许 |
| `89` | 更正公告公示 | 条件允许，必须做生命周期与有效期复核 |
| `91` | 中标候选人公示 | 拒绝 |
| `90` | 中标结果公示 | 拒绝 |

分类证据来自入口页的原生分类控件，而不是根据公告标题推断：[官方入口](https://bulletin.cebpubservice.com/)。

### 1.2 已核验列表 URL

平台入口通过 iframe 加载分类列表，当前公开路径如下：

- 招标公告：`https://bulletin.cebpubservice.com/xxfbcmses/search/bulletin.html`
- 资格预审公告：`https://bulletin.cebpubservice.com/xxfbcmses/search/qualify.html`
- 更正公告：`https://bulletin.cebpubservice.com/xxfbcmses/search/change.html`
- 明确排除的中标结果：`.../search/result.html`
- 明确排除的中标候选人：`.../search/candidate.html`

可复核样例：

- [招标公告列表（7 日、第一页）](https://bulletin.cebpubservice.com/xxfbcmses/search/bulletin.html?dates=7&categoryId=88&page=1&showStatus=1)
- [资格预审公告列表（7 日、第一页）](https://bulletin.cebpubservice.com/xxfbcmses/search/qualify.html?dates=7&categoryId=92&page=1&showStatus=1)
- [更正公告列表（7 日、第一页）](https://bulletin.cebpubservice.com/xxfbcmses/search/change.html?dates=7&categoryId=89&page=1&showStatus=1)

列表脚本当前支持的查询参数包括：

| 参数 | 页面用途 |
|---|---|
| `categoryId` | 原生公告类别 |
| `page` | 页码，从 1 开始 |
| `dates` | 相对发布时间范围；页面使用 `0/2/3/7/30/90/300` |
| `searchDate` | 相对时间计算后的日期 |
| `startcheckDate`, `endcheckDate` | 明确的公告发布时间范围 |
| `industryName` | 行业 |
| `area` | 地区 |
| `publishMedia` | 发布媒介 |
| `sourceInfo` | 来源渠道 |
| `word` | 关键词；页面脚本执行双重 URI 编码 |
| `status` / `signDate` | “有效期内”控件使用的状态/时点参数 |
| `showStatus` | 页面来源展示方式 |

入口页“有效期内”单选框调用 `searchBysignUp('01')`。列表脚本随后附加当前时点形式的 `signDate=<now>,lt`。这里的 `lt` 是平台后端自己的查询语法，不能自行按数据库比较符解释；实现应复现平台控件行为，同时仍在本地校验解析出的实际截止时间。[官方入口及其公开脚本](https://bulletin.cebpubservice.com/)

### 1.3 列表字段、时间与分页

招标公告和资格预审列表当前每页 15 条。列表行直接提供：

- 公告 UUID（`urlOpen(uuid)`）；
- 名称；
- 行业；
- 地区；
- 来源渠道；
- 公告发布日期；
- `openTime` 的完整时间戳。

招标公告列表把最后一列标为“距离开标时间”，资格预审列表标为“距离开启时间”；页面脚本用该时间戳计算“剩余 N 天/已开标”。这比从标题推断是否有效可靠，但仍应把它作为“截止/开启时间候选字段”，因为不同上游交易平台的数据语义可能不完全一致。

详情跳转由列表脚本生成：

`https://ctbpsp.com/#/bulletinDetail?uuid=<uuid>&inpvalue=&dataSource=0&tenderAgency=`

分页由同一列表 URL 的 `page` 参数完成。页面在大量结果时把末页限制为 500、总显示条数限制为 10000；因此历史回填不能单次从第 1 页一直翻到底，应按较短发布时间区间分片，避免静默漏数。

### 1.4 更正公告的关键风险

更正列表提供“更正公告公示名称”和“原公告公示名称”，两者可用于建立变更到原公告的关系。但本次实时样本中，同一 `categoryId=89` 列表明确出现了流标和终止消息；因此：

- `categoryId=89` 只证明它是平台归入“更正公告公示”的事件；
- 它不证明项目仍可报名或投标；
- 全国入口页在更正分类下还会隐藏“有效期内”控件，不能指望网站侧替我们完成有效期筛选。

更正公告必须在取得详情、关联原公告并计算最新有效截止时间后才可输出。若详情反自动化挑战未通过、原公告关联缺失、正文不能解析出有效时间，必须安全失败：记录抓取诊断，但不向用户展示为招标机会。

### 1.5 访问约束

- 2026-07-15 实测三个列表路径无需登录即可返回 HTML。
- 公开脚本保留了 VAPTCHA/验证码和 `VerificationCode` 路径，平台可以随时启用人机验证；采集器不得绕过验证码。
- 列表详情应用 [ctbpsp.com](https://ctbpsp.com/) 是 JavaScript SPA。其公开脚本引用了 `cutominfoapi/bulletinuuid/<uuid>`，但直接请求该内部接口时返回反自动化 JavaScript 挑战，而不是可依赖的 JSON。因此不能把该未公开内部接口作为生产契约。
- 安全策略：检测验证码/挑战页/非预期 Content-Type/缺少表头时返回“来源暂不可用”，不得把挑战 HTML 当公告正文。

## 2. 上海市公共资源交易平台（上海）

### 2.1 建议的第一阶段范围

第一阶段建议接入上海总平台中的“国企采购”官方栏目：[上海市公共资源交易平台—国企采购](https://www.shggzy.com/gqcg)。该栏目在同一官方域名下提供清晰的三类目标信息映射。

原生栏目 ID 和标签为：

| `channelId` | 平台标签 | 本项目处理 |
|---|---|---|
| `2662` | 招标公告/资格预审公告 | 允许，但须在正文中结构化细分两类 |
| `2663` | 变更公告 | 条件允许，须关联原公告并复核有效期 |
| `2664` | 中标候选人公示 | 拒绝 |
| `2665` | 中标结果公告 | 拒绝 |
| `2666` | 异常公告 | 拒绝 |
| `2661` | 招标计划/采购意向 | 当前需求拒绝 |
| `2671` | 年度工作计划 | 当前需求拒绝 |

分类证据：[国企采购栏目](https://www.shggzy.com/gqcg)。

上海“建设工程”官方栏目另有 `channelId=29`，标签是“招标公告和资格预审公告”，候选人和结果分别是 `32` 和 `33`：[建设工程栏目](https://www.shggzy.com/jyxxgc)。但已核验页面没有独立的建设工程变更公告栏目，而且 `29` 同样把招标与资格预审合并。为了满足“只输出三类且类型可证明”的严格要求，建议第一阶段只启用国企采购 `2662/2663`；建设工程及其他分平台应在逐个验证分类映射后再启用。

### 2.2 列表、检索和分页

已核验列表：

- [招标公告/资格预审公告](https://www.shggzy.com/search/queryContents.jhtml?channelId=2662)
- [变更公告](https://www.shggzy.com/search/queryContents.jhtml?channelId=2663)

查询表单使用 GET，请求字段包括：

`title`, `channelId`, `origin`, `inDates`, `timeBegin`, `timeEnd`, `ext`, `ext1`, `ext2`, `ext3`, `cExt`。

页面提供的相对发布时间选项为 `1/3/7/30/90/365` 天，也允许 `timeBegin` 和 `timeEnd`。分页脚本显示每页 10 条，最多展示 2500 页。后续页的当前形态为：

`/search/queryContents_<page>.jhtml?cExt=1&channelId=<channelId>&...`

实测参数顺序/表单参数不完整时，服务器可能返回 HTTP 200 但正文长度为 0。采集器必须把“200 + 空正文”视为失败，不能视为“没有公告”。

招标/资格预审详情当前使用类似路径：

`https://www.shggzy.com/gqcgzbgg/<content-id>`

公开样例：

- [上海银行全面风险领域能力建设专家人力资源采购项目](https://www.shggzy.com/gqcgzbgg/8939180)
- [上海农商银行信创 GPU 服务器购置](https://www.shggzy.com/gqcgzbgg/8921959)

本次 `2663` 实时列表没有返回可用于验证的变更详情样本，因此不能宣称已确认变更详情 URL 前缀或固定 DOM 结构。实现必须从列表返回的真实链接跟随，不能猜测 URL。

### 2.3 可用于有效期判断的正文结构

已核验详情页虽然是公开 HTML，但关键时间目前主要在公告正文中，而不是稳定 JSON 字段。样例中可稳定锚定的章节/字段包括：

- `招标文件的获取` → `文件发售时间` 或 `获取时间`；
- `投标文件递交` / `投标文件的递交` → `递交截止时间`；
- `开标时间`；
- 详情页元数据中的 `项目编号`、`发布时间`、`信息来源`；
- 页面面包屑和正文上方的原生栏目标签“招标公告/资格预审公告”。

“我要参与”、招标文件下载、投标文件递交需要登录/CA；读取公开公告列表和正文在本次实测中不需要登录。采集器只应读取公开公告，不得自动登录或绕过 CA 流程。

## 3. 推荐的公告类型与“仍可投标”判定算法

### 3.1 第一层：来源与分类白名单（主要证据）

每条候选记录必须保存不可变的 `source_category_id`、列表 URL、详情 URL、抓取时点和页面原生分类标签。

- 全国只请求 `88/92/89`；永不请求 `90/91` 作为机会来源。
- 上海国企采购只请求 `2662/2663`；永不请求 `2664/2665/2666` 作为机会来源。
- 响应页面的表头、面包屑或栏目标签必须与请求分类一致；不一致则隔离。

这一步直接消除了绝大多数中标候选人和中标结果，无需猜标题。

### 3.2 第二层：正文“结构语法”细分，不以标题为主

全国 `88` 与 `92` 已由来源原生分类区分，可直接标准化为 `TENDER_NOTICE` 和 `PREQUALIFICATION_NOTICE`。

上海 `2662` 合并了两类，建议使用成组结构字段分类：

- 资格预审：正文同时出现资格预审文件获取章节，以及资格预审申请文件递交/截止章节；
- 招标公告：正文同时出现招标文件获取章节，以及投标文件递交/投标截止/开标章节；
- 两组都不完整、两组互相冲突或正文只有附件无法读取：标记 `AMBIGUOUS`，不向用户输出。

标题中的“招标公告”“资格预审公告”只能作一致性校验或报警信号，不能作为唯一分类依据。

实现时应把章节标题作为锚点，在相邻章节边界内提取字段，不能在整篇正文里用一个宽松正则找任意“截止时间”，否则容易把合同履行期、业绩期限或历史日期误当投标截止时间。

### 3.3 第三层：有效参与窗口

在 `Asia/Shanghai` 时区规范化以下字段：

- 文件获取/报名结束时间；
- 资格预审申请文件递交截止时间；
- 投标文件递交截止时间；
- 开标/开启时间（只作交叉校验，不能无条件代替递交截止时间）。

规则：至少一个可操作的报名、资格预审申请或投标递交窗口在当前时点之后，且没有更高优先级的终止事件，才标记 `ACTIVE`。日期缺少年份、只有“另行通知”、解析冲突或时区不明时，不猜测，进入人工复核队列。

全国列表的 `openTime` 和官网“有效期内”筛选可作为第一道筛选，但本地仍必须精确比较时间；上海从正文锚定章节提取时间。

### 3.4 第四层：更正公告采用事件链，而不是单条文本分类

更正/变更必须满足全部条件：

1. 来源分类是全国 `89` 或上海 `2663`；
2. 通过平台提供的原公告链接、项目编号、招标编号或标段编号关联到一条已确认的招标/资格预审原公告；
3. 按发布时间顺序把变更覆盖到原公告字段，得到最新的文件获取、申请递交和投标截止时间；
4. 正文的操作性内容没有把项目状态改为终止、取消、流标、废标；
5. 最新有效参与截止时间仍晚于当前时点。

终止判断也不应只扫标题。应在正文的“变更内容/公告内容/终止原因”等操作性章节中识别明确的生命周期动作，并结合是否给出新的未来截止时间。若只能读到标题、无法取得正文或无法关联原公告，则安全失败，不输出。

### 3.5 推荐决策模型

```text
ALLOW = source_category_is_allowlisted
     AND response_category_matches_request
     AND body_structure_classifies_to_one_allowed_type
     AND required_identity_fields_present
     AND active_deadline_is_parsed_and_future
     AND no_later_terminal_event

otherwise => QUARANTINE / REJECT, never silently downgrade to title matching
```

建议同时保存 `classification_evidence`，例如：来源类别、命中的章节、提取的原文时间片段、关联原公告 ID、最新变更 ID和拒绝原因，便于在产品中向用户解释“为什么这是仍可参与的招标”。

## 4. 实施时的安全失败要求

- 分类 ID、栏目表头、详情正文、项目标识或有效截止时间任一关键证据缺失：不展示。
- HTTP 200 但空正文、验证码页、反自动化脚本页、登录页、Content-Type 异常：标记来源故障，不当作零结果。
- 不绕过验证码，不自动登录，不写入 Cookie/账号密码，不调用未公开内部接口作为稳定生产契约。
- 页面 DOM 或分类 ID 漂移时立即告警并停止该来源，避免把中标结果误投放给用户。
- 每个来源保留离线 HTML 契约样本和分类回归测试；测试必须包含候选人、结果、异常、终止/流标等负样本。

## 5. 尚未验证、不得假设的事项

- 全国详情 SPA 的内部 JSON 返回字段及稳定性未通过反自动化挑战验证。
- 全国更正公告详情中的原公告关联字段和变更后截止时间结构未形成可靠契约。
- 上海 `2663` 在研究时没有实时列表样本，因此变更详情 URL 前缀和 DOM 尚未验证。
- 上海建设工程及其他分平台没有统一的三类公告分类方案；不能把国企采购的 `2662/2663` 映射复用到其他分平台。
- 两站均未核验到面向第三方、带版本和服务承诺的公开数据 API。

以上未验证项应保持功能关闭或隔离，直到取得真实官方样本并补齐契约测试。
