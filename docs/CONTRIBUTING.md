# 双人协作规范

## 分支与评审

- 默认分支 `main`，禁止直接推送。
- 分支使用 `feat/issue-编号-名称`、`fix/issue-编号-名称`、`docs/issue-编号-名称`。
- 每个 PR 关联 Issue，列出验收标准、测试结果和截图，由另一名队员审核。
- 每人同时最多一个 In Progress Issue；每周结束保持 `main` 可运行。

## 责任边界

- 数据科学/产品负责人：字段字典、相关性、实体匹配、评测集、证据覆盖、第二来源。
- 软件工程负责人：API、调度、存储、第一来源、前端、Word、CI 与稳定性。
- `JobSpec`、`TenderRecord`、`SourceAdapter` 是共同契约，修改须双方审核。

## 完成定义

代码、测试、文档和失败路径同时完成才算 Done。CI 不得访问真实网站；测试数据必须脱敏或来自明确公开样例。
